# 제목
소수 찾기

### 문제
주어진 수 N개 중에서 소수가 몇 개인지 찾아서 출력하는 프로그램을 작성하시오.

### 입력
첫 줄에 수의 개수 N이 주어진다. N은 100이하이다. 다음으로 N개의 수가 주어지는데 수는 1,000 이하의 자연수이다.

### 출력
주어진 수들 중 소수의 개수를 출력한다.

## 풀이 핵심 아이디어
에라스토테네스의 체를 활용한 방법으로 문제에서 주어진 자연수의 범위는 1,000이하의 자연수이기 때문에 1000개의 소수를 구하는것이고 사실 시간복잡도를 생각하면 상수개의 소수를 구하는것이기 때문에 O(1)의 시간복잡도를 가지게 된다, 즉 해당 문제를 풀이한 가장 핵심 아이디어는 최대 범위갯수의 소수를 모두 구하여 입력받은 수가 소수인지 체킹하는방법으로 문제를 풀이했다. 

### 1. 요구사항
N개의 수가 주어지고 N개의 숫자중 소수가 몇개인지 찾아 출력하시오.

### 2. 구현을 위해 필요한 정보
1. N ≤ 100
2. 주어지는 수 ≤ 1,000

### 3. 접근 아이디어
해당 문제를 풀이한 아이디어는 에라스토테네스의 체 알고리즘이 가장 중요한 아이디어라 생각합니다, 해당 알고리즘을 이해하고 있다면 정해진 범위내의 모든 소수를 구할 수 있으며 
주어진 수들이 해당 인덱스 자리의 수가 소수인지 판별하는 인덱스 번호가 되기 때문에 해당 인덱스가 true라면 소수이고 아니라면 false이기때문에 간단히 문제를 풀이할 수 있다.

### 4. 구현 과정
입력받을 수의 갯수를 입력받는다, 주어지는 수를 입력받는다, 그 후 에러스토테네스의 체 알고리즘을 통해, 2~1000까지의 숫자들 중 소수를 구한다, 입력받은 수가 인덱스번호가 되어 해당 인덱스가 true라면 count의 값을 1증가시켜준다.


### 5. 체크 사항
만약 주어진 범위가 상수가 아니라면 해당 문제를 풀이하는 방법에 대해서 다시한번 고민해보았을것같다,
정해진 범위내의 소수를 모두 올바르게 구했는지에 대해서 잘 체크해보아야한다.

### 6. 최적화
해당 문제는 정해진 범위내의 소수를 구하는 문제이기 때문에 더이상의 최적화는 필요하지 않다고 판단했습니다.


### 7. 시간복잡도
O(N)
